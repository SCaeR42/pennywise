
# PennyWise (веб приложение для учёта финансов)

## План реализации VUE.js

План реализации: Проект "Учёт финансов" (Vue 3 Edition)

### Технологический стек

|**Аспект**|**Решение (Vue)**|
|---|---|
|**Framework**|Vue 3 (Script Setup + Composition API)|
|**Build Tool**|Vite|
|**State Management**|Pinia (с persist-плагином для localStorage)|
|**Forms**|VeeValidate + Zod (или FormKit)|
|**Routing**|Vue Router 4|
|**UI Kit**|shadcn-vue + Tailwind CSS|
|**PWA**|vite-plugin-pwa (не входит в MVP)|

### Этапы реализации

#### Этап 1: Базовая структура и PWA

- Инициализация проекта через `npm create vue@latest`.
- Настройка **Tailwind CSS** и **shadcn-vue**.
- Настройка `vite-plugin-pwa` для работы в offline-режиме.
- Настройка маршрутизации (Vue Router) с использованием `meta: { requiresAuth: true }` для защиты путей.

#### Этап 2: Промо-страница

- Создание View `LandingView.vue`.
- Использование компонентов shadcn для Hero-секции и Accordion (FAQ).

#### Этап 3: Авторизация (Auth Store)

- **Pinia Store:** Создание `useAuthStore`. Состояние: `user`, `isAuthenticated`.
- **Middleware:** Использование `router.beforeEach` для проверки прав доступа.
- _Важно:_ На данном этапе используем mock-данные в `localStorage`.

#### Этап 4: Транзакции

- **Формы:** VeeValidate в связке с Zod (опционально, не в MVP) — это стандарт индустрии Vue для валидации.
- **Список:** Таблица (shadcn `Table`) с использованием `computed` свойств для фильтрации и пагинации.
- _Примечание:_ Сверка с [документацией VeeValidate](https://vee-validate.logaretm.com/v4/) обязательна при реализации сложных форм.

#### Этап 5: CRUD справочников

- Реализация через Pinia Stores (один store на сущность, например `useAccountStore`).
- Использование реактивных массивов (`ref` или `reactive`).

#### Этап 6: Настройки и Тёмная тема

- **Тема:** Использование `useDark` и `useToggle` из библиотеки **VueUse**. Это наиболее актуальный подход для управления темами во Vue.
- Сохранение настроек пользователя в `localStorage`.

## Ключевые рекомендации по инструментам

1. **Управление состоянием:** Pinia является официальным решением. Используйте `pinia-plugin-persistedstate` для автоматической синхронизации с `localStorage` — это избавит вас от написания лишнего кода для сохранения данных.
2. **Валидация форм:** При работе с Vue я рекомендую **VeeValidate**. Она нативно интегрируется с Vue и отлично работает с библиотекой схем **Zod**.
3. **Утилиты:** Обязательно подключите [VueUse](https://vueuse.org/). Это "швейцарский нож" для Vue-разработчика (управление темой, отслеживание событий окна, локальное хранилище, и т.д.).


## Структура презентации
### 1. Цели проекта

- **Основная задача:** Создание удобного, кроссплатформенного инструмента для контроля личных финансов.

- **Ключевые цели:**
    - Обеспечение приватности (локальное хранение данных).
    - Высокая скорость работы за счёт PWA.
    - Интуитивный интерфейс для быстрого ввода транзакций.

### 2. Что планировалось

- Реализация полного цикла управления бюджетом: от учёта доходов/расходов до аналитики.
- Разделение сущностей (счета, категории, теги).
- Адаптивность (Mobile-first подход).
- Масштабируемость: переход от `localStorage` к облачному бэкенду (Supabase).

### 3. Используемые технологии

- **Frontend:** Vue 3 (Composition API), Vite.
- **Управление состоянием:** Pinia (Persistence plugin).
- **Маршрутизация:** Vue Router 4.
- **UI/UX:** Tailwind CSS, shadcn-vue.
- **Валидация:** VeeValidate.

### 4. Что получилось

- **Функционал:** Полный CRUD справочников и транзакций, фильтрация данных.
- **UX:** Реализована «тёмная тема» (через VueUse), sidebar-навигация.
- **Производительность:** Быстрая загрузка и работа приложения без сетевого соединения (PWA).

### 5. Схемы / Архитектура

- _Здесь важно показать потоки данных._ * **Архитектурная схема:** Клиент (Vue SPA) → Pinia Store → Storage (LocalStorage/Supabase).

### 6. Выводы

- **Результат:** Готовый MVP, готовый к деплою как PWA.
- **Уроки:** Выбранный стек (Vue 3 + Pinia) обеспечивает высокую скорость разработки и удобство поддержки кода.
- **Планы:** Интеграция с банковскими API, расширенная аналитика (графики), переход на Supabase для синхронизации между устройствами.
